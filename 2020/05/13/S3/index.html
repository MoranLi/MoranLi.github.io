S3

S3

  • object-oriented storage
    • files 0 - 5TB
    • ulimit storage
    • file are store in bucket
    • bucket are private when created
      • S3 is a univerisal namspace, all url to s3 busket mush be unique
      • Object in S3
    • key (name of file)
    • value (content of file)
    • version ID
    • metadata
    • subresources
      • access control lists
      • torrents
        • data consistincy
    • read after write
    • eventual consistency for overide delete & write
    • can use MFA delete to product delete
      • S3 guarantees:
    • 99.9% availability
    • 99.999999999% durability
      • Storage Class (expensive -> cheap)
    • standard
    • Intelligent Tiering
      • automaticly move data between tire to minimum cost without influence performace
    • IA - infrequently accessed
    • One Zone IA
      • IA without data consistency across differenct availability zone
    • Glacier
      • can store any amount of data
      • used on archieved data
      • require minutes to hours to retrival data
        • Glacier Deep Archive
        • lowest cost
        • about 12 hour retrival time
        • cross-region replication
        • replicate data from one region to another
        • file in exist bucket would not replicate automatically
        • all update after will update automatic
        • delete / delete of indidval version would not replicate
        • transfer acceleration
        • user transfer file to edge location and take advantage of using network between S3 data center location and edge location
        • security
        • set access with
        • bucket policy
        • access control list
        • can log request
        • encryption
        • During transmission
        • SSL/TLS
        • server side
        • SSE-S3 sever side encryption S3
        • SSE-KMS Key management service
        • SSE-C sever side encryption with consumer provided keys
        • client side encryption
        • versioning
        • can not be disabled once enabled
        • store all version even delete (label with deleted)
        • intergated with lifecycle rules
          • MFA deletion capacity, provide additional layer of security
        • lifecycle
        • automate move object to different layers
        • can use conjunction with verisoning
        • can apply to current / past versioning
        • share bucket between account
        • use bucket policy & IAM(apply to entire bucket), programmatic only
        • use bucket ACL & IAM(individual objects), programmatic only
        • cross-account IAM roles, console & programmetic
        • cloudfront
        • deliver your content to nearest edge location for user for best possible performance
        • origin: origin of all the file CDN will distribute (S3, EC2, load balancer, Route 53)
        • distribution: nmae of edge location give to CDN for distribute
        • object are cached for the life of TTL(time to live)
        • clear cached is charged
        • Web distribution
        • RTMP (media streaming)
        • snowball
        • PB level data transport solution using physical secure large disk
        • import / export to S3
        • storage gateway
        • File gateway
        • NFS, store directly on S3
        • volume gateway
        • stored volume
          • entire dataset store on site and asycgriniusky backup to S3
        • cached volume
          • entire dataset store on S3 and most frequenctly accessed cached on site
        • gateway virtual library
        • use for backup
        • Athene
        • Interactive query service
        • allow you to query data located in S3 using standard SQL
        • serverless
        • commonly used to analyse log data in S3
        • Macie
        • security service use AI to discover, classify and protect data in S3 and help identify PII(Personal identifiable information)
        • also use to analyze cloud trail logs for suspicious API activity
        • include dashboard, report and alerting
        • great for PCI-DSS compliance and preventing ID theft
