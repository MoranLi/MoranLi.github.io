EC2

  • EC2
    • virtal machine provide resizable compute capicity, reduce time to deploy new server instance
    • will be charged for a partical hour of usage
    • Pricing Type
      • on demand
        • fix rate / time
      • reserved
        • provided contract of a capicity reservation for term of 1 - 3 years
      • spot
        • bid for what price want for capicity, save more for fix start / end time
        • won`t charge for stop running instance
      • dedicated hosts
        • physical ec2 instance, allow you use existing sever-bound software licenses
    • Instance Type
    • new instance
      • termiantion protection is turned off by default
      • root drive / additional drive can be encrypted
        • os level encryption
        • make a snapshot, encrypt it, create a new AMI use that snapshot
        • can be done throught cli, console or API
      • root dirve is deleted when terminate by default
    • security group
      • change to security group effect to instance immediatly
      • all inbound traffic is blocked by default
      • all outbound traffic is allowed
      • security group have many to many relationship to ec2 instance
      • stateful
        • enable inbound, outbound enable automatic
      • can not block specific IP address using security group, but with Network Access Lists
      • only specify allow rules not deny rules, but with Network Access Lists
    • EBS
      • SSD (can used as root drive)
        • General Purpose SSD
        • Provision IOPS SSD
          • Highest performance
          • use for database
      • HDD
        • Throughput Optimized HDD
        • COLD HDD
        • Magnetic (can used as root drive)
    • Volume & Snapshot
      • Volume: disk store EC2 instance exits in EBS
        • can change size/storage type when running
        • same available zone with EC2
        • move EC2 between AZ / region:
          • make snapshot, copy to another AZ/ region, use it to launch new instance
      • Snapshot: a version(time copy) of Volume exist in S3
        • incremental: only block change updated in snapshot
        • better to stop instance and take snapshot
        • can use for recreate EC2 instance
    • AMI Type
      • EBS volume
        • root device of an instance from AMI create by a EBS snapshot
        • can keep even terminate
      • Instance Store Volume
        • same as emphemeral storage
        • root device of an instance from AMI create by a templayte stored in S3
        • can not stop or you will lose all data
        • reboot whould not lose data
    • Elastic Load balancer
      • Type
        • Application Load Balancer
          • best suit for HTTP / HTTPS request
          • layer 7
        • Network Load Balancer
          • best suit for TCP traffic
          • layer 4
          • use for extreme perfirmance
          • most costly
        • Classic Load Balancer
          • both feature of Application and Network Load Balancer
          • 504 error if app not working
      • add X-Forward_For header to convert IPV4 private address to IPV4 public address
    • Route53
      • DNS service
      • map domain name to
        • EC2
        • load balancer
        • S3
    • RDS
      • OLTPs
        • SQL
          • MYSQL, POSTGRESQL, ORACLE, AURORA, MARAIA DB
        • NOSQL
          • DynameDB
      • OLAP
        • redshift
      • memory caching
        • memchched
        • redis
      • Multiple AZ
        • only for diaester recovery
      • Read replica
        • inprove performance
        • for scaling, not for DR
        • must have auto backup turn on
        • up to 5 read replica / db
        • have read replica for read replica (have altency problem)
        • can use with multiple AZ
        • read replica -> own DB, this break repication
        • MYSQL and MARIZDB can have read replica in another region
    • CLI
      • you can use AWS client on your PC
    • Elastic Cache
      • cache data usually uesd in memory, reduce time complexity of read / write from disk
      • use redshift fro OLAP heavy, thie is for OLTP
      • use if
        • caching object
        • keep things simple
        • scale cache horizontally
      • use redis if
        • have advanced data type (list, set, hash)
        • doing data sorting / ranking
        • data persistance
        • multi AZ
        • publisher / subscripter capabilities are needed
