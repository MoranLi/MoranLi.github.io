S3

S3

  • object-oriented storage
    • files 0 - 5TB
    • ulimit storage
    • file are store in bucket
  • S3 is a univerisal namspace, all url to s3 busket mush be unique
  • Object in S3
    • key (name of file)
    • value (content of file)
    • version ID
    • metadata
    • subresources
      • access control lists
      • torrents
  • data consistincy
    • read after write
    • eventual consistency for overide delete & write
    • can use MFA delete to product delete
  • S3 guarantees:
    • 99.9% availability
    • 99.999999999% durability
  • Storage Class (expensive -> cheap)
    • standard
    • Intelligent Tiering
      • automaticly move data between tire to minimum cost without influence performace
    • IA - infrequently accessed
    • One Zone IA
      • IA without data consistency across differenct availability zone
    • Glacier
      • can store any amount of data
      • used on archieved data
      • require minutes to hours to retrival data
        • Glacier Deep Archive
        • lowest cost
        • about 12 hour retrival time
        • cross-region replication
        • replicate data from one region to another
        • transfer acceleration
        • user transfer file to edge location and take advantage of using network between S3 data center location and edge location

Overview and IAM

10000 Foot Overview Of AWS

  • region
    • a phvsical location contain 2 or more AZs
  • availability zone
    • one or more data centers, each have seperate redunent power, networking and connectivity, hosed in seperated faciliaties
  • edge location
    • end point used to caching AWS content

IAM

  • IAM is universial, you are create a user globally
  • root account is the account first setup AWS account, it have full admin access
  • new user have no permission when first created
  • new user assigned access key ID and secret access key
  • access key ID & secret access key is not same as password, it can used to access service through API / command line but can not use to login to console
  • you can only view access key ID & secret access key once, so store it somewhere locally, if it is lose, admin can regenerate it
  • always setup MFA for root account
  • you can setup own password policy
